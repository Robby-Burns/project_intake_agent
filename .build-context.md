## 🛠️ Important Architectural Decisions

**Decision:** Agnostic Factory Pattern
**Why:** To prevent vendor lock-in. We can now swap OpenAI for Anthropic or Azure by changing `scale.yaml` and `llm_factory.py`, without touching agent code.

**Decision:** Hybrid API/UI Architecture
**Why:** The framework requires an API for Audit webhooks (`POST /audit/*`), but the user interface is Streamlit. We deployed both as separate services in `docker-compose` to satisfy both requirements without bloating the UI code.

**Decision:** Database Persistence
**Why:** The original in-memory list `conversation_history` risked data loss. We moved to a `DatabaseAdapter` pattern to ensure every turn is saved immediately.
