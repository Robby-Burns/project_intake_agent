## ✅ Resolved Bugs
### BUGS-002: ModuleNotFoundError in Docker UI
**Status:** Resolved | **Severity:** Critical (Blocker)
**Description:** `streamlit run app/ui.py` failed with `ModuleNotFoundError: No module named 'app'` or `No module named 'msal'`.
**Root Cause:** Multiple issues:
1.  `PYTHONPATH` was not set in the Docker container.
2.  `msal` dependency was missing from `pyproject.toml`.
3.  `app/pm_tools/__init__.py` was eagerly importing all adapters, causing crashes even if the tool wasn't being used.
**Fix:**
1.  Added `ENV PYTHONPATH=/app` to `Dockerfile`.
2.  Added `msal` to `pyproject.toml`.
3.  Refactored `app/pm_tools/__init__.py` to only expose the factory, enforcing lazy loading.
4.  Added validation to the `PMToolFactory` to provide clear errors if environment variables are missing.

## 🔍 Bug Patterns Identified (AI-UPDATED)
*(AI: Read this before writing new integration code to avoid repeating mistakes)*

### Pattern 1: Sidecar Race Conditions
**Occurrences:** 2 times (MCP filesystem + Database)
**Root cause:** Containers starting before services are ready
**Solution:** Always use exponential backoff decorators on connection logic.

### Pattern 2: API Rate Limiting
**Occurrences:** 1 time (External Search Tool)
**Root cause:** Parallel agent execution triggered 429 errors.
**Solution:** Implemented the Token Bucket algorithm in the `RateLimiter` middleware.

### Pattern 3: Docker Import Paths
**Occurrences:** 1 time (Streamlit UI)
**Root cause:** Running scripts from subdirectories in Docker breaks absolute imports if `PYTHONPATH` is not set.
**Solution:** Always set `ENV PYTHONPATH=/app` (or workspace root) in Dockerfiles for Python projects.

### Pattern 4: Undeclared Dependencies
**Occurrences:** 1 time (`msal` for Planner)
**Root cause:** A dependency was used in the code but not declared in `pyproject.toml`.
**Solution:** All `import` statements for third-party libraries must have a corresponding entry in `pyproject.toml`.
