# 🏗️ System Architecture - ProjectIntakeAgentTwo

> **Status:** Complete
> **Pattern:** Parallel Orchestrator ("Whisper Engine")
> **Risk Level:** Medium (Score: 8)

---

## 1. High-Level Design

The system uses a **Parallel Orchestrator Pattern** to enable real-time interviewing with multi-agent analysis. Instead of a sequential chain, 8 specialist agents run in parallel (asynchronously) to analyze user input and "whisper" questions to the main Interviewer Agent.

```mermaid

    PDF --> |Save to reports/| Storage[File System]
    Orchestrator --> |Create Ticket + Attach PDF| PMTool[Monday/Jira Adapter]
```

## 2. Core Components

### A. The Security Layer (Hybrid Guardrails)
*   **Role:** Intercepts all messages before they reach the LLM.
*   **Implementation:** **Hybrid Approach** (Regex + NLP).
    *   **Regex:** Strict patterns for SSN, Credit Cards, Member IDs.
    *   **Presidio (NLP):** Context-aware detection for Names, Phones, Emails.
    *   **Configurable Model:** Uses `en_core_web_sm` (Local) or `en_core_web_lg` (Prod).

### B. The Interviewer Agent (Synthesizer)
*   **Role:** The "Face" of the system. Maintains natural conversation.
*   **Responsibility:**
    *   Receives user input.
    *   Receives "whispered" questions from Specialists.
    *   **Decision Logic:** Selects the *single most relevant* question based on **Priority Score (1-10)**.
    *   Queues other questions for the final report.

### C. The Specialist Swarm (8 Agents)
*   **Execution:** Asynchronous / Parallel.
*   **Session Strategy:** **Stateless** (Fresh instances per session).
*   **Model Strategy:**
    *   **Complex Agents (GPT-4o):** InfoSec, IT, ERM, Project Manager, Product Manager.
    *   **Simple Agents (GPT-4o-mini):** Marketing, Training, Accounting.
*   **Capabilities:**
    *   **Real-time:** Whisper questions during chat.
    *   **Post-process:** Generate Domain Summaries at session end.

### D. The PDF Generator
*   **Trigger:** End of interview.
*   **Output:** Saved to `reports/` directory.
*   **Content:**
    1.  **Metadata:** Project Name, User, VP#, Timestamp.
    2.  **Executive Summary:** Generated by LLM.
    3.  **Key Findings:** Bulleted list of takeaways.
    4.  **Specialist Sections:** Domain Summary + Top 5 Critical Questions per agent.
    5.  **Full Transcript:** Color-coded chat log.

### E. PM Tool Integration
*   **Pattern:** Adapter Pattern.
*   **Supported Tools:** Monday.com, Jira, Mock.
*   **Functionality:** Creates ticket, adds description, uploads PDF (using tool-specific API logic).

---

## 3. Data Flow

1.  **Input:** User sends message via Streamlit UI.
2.  **Sanitization:** Hybrid Guardrails strip PII.
3.  **Broadcast:** Interviewer sends text to all 8 specialists.
4.  **Analysis (Async):** Specialists analyze and whisper questions.
5.  **Synthesis:** Interviewer selects best question.
6.  **Response:** Interviewer responds to user.
7.  **Finalization:**
    *   Orchestrator triggers parallel summary generation.
    *   PDF Generator compiles the report to `reports/`.
    *   PM Tool Adapter uploads the report to Monday/Jira.

## 4. Tech Stack

| Component | Technology | Reasoning |
|-----------|------------|-----------|
| **UI** | **Streamlit** | Rapid, interactive web interface. |
| **Orchestration** | **LangGraph** | Best for stateful, cyclic multi-agent flows. |
| **LLM** | **OpenAI GPT-4o / Mini** | Cost/Performance optimization. |
| **Async** | **Python `asyncio`** | Required to run 8 agents without latency lag. |
| **Security** | **Microsoft Presidio** | Enterprise-grade PII detection. |
| **PDF** | **ReportLab** | Programmatic PDF generation. |
| **Deployment** | **Docker + uv** | Fast, reproducible builds. |

---

## 5. Scalability & Performance

*   **Latency Target:** < 3 seconds per response.
*   **Optimization:** Specialists run in parallel.
*   **Docker:** Uses `uv` for fast dependency installation.
